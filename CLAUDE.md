# CLAUDE.md

## コンテキスト管理（最重要）
- コンテキストウィンドウは最も重要なリソース。満杯になるとパフォーマンスが低下する
- 関連のないタスク間では `/clear` でコンテキストをリセットする
- 調査・探索はサブエージェントに委譲し、メインコンテキストを汚染しない
- スコープを絞らない「調査して」は禁止。必ず範囲を限定する

## 作業の進め方
1. **探索**: Plan Modeでファイルを読み、現状を理解する
2. **計画**: 実装計画を作成する
3. **実装**: Normal Modeでコーディングし、計画に対して検証する
4. **コミット**: 説明的なメッセージでコミット

小さな修正（タイポ、変数名変更など）は計画をスキップして直接実行してよい。

## 自己検証
- IMPORTANT: テスト、リンター、ビルド、スクリーンショット等で必ず自分の作業を検証する
- 検証できないものは出荷しない
- 症状ではなく根本原因に対処する。エラーを抑制しない

## プロンプトのコツ
- 具体的なファイル、制約、パターンを指定する
- 既存コードのパターンを参照して「これに従え」と指示する
- `@` でファイル参照、画像貼り付け、URL指定でリッチコンテキストを提供する
- 大きな機能は最初にインタビュー形式で要件を詰める

## セッション管理
- 2回修正しても直らない場合 → `/clear` して学びを反映した新しいプロンプトで再開
- `Esc` で中断、`Esc+Esc` または `/rewind` で巻き戻し
- `claude --continue` で中断したセッションを再開
- チェックポイントで安全に実験し、失敗したら巻き戻す

## 避けるべきパターン
- 1セッションで無関係なタスクを混在させる（キッチンシンク）
- 失敗した修正を何度も繰り返す（コンテキスト汚染）
- CLAUDE.mdを長くしすぎる（重要なルールがノイズに埋もれる）
- 検証なしで出力を信頼する
- スコープなしの無限探索

## スケーリング
- `claude -p "prompt"` で非対話型実行（CI、スクリプト統合）
- 複数セッション並列実行で開発を高速化
- Writer/Reviewerパターン: 1つのセッションで実装、別のセッションでレビュー
- ファンアウト: ファイルリストに対してループで `claude -p` を実行

## 環境拡張
- CLIツール（`gh`, `aws`等）をインストールして外部サービスと連携
- MCP サーバーで Notion, Figma, DB 等を接続
- フックで毎回実行すべきアクション（lint等）を自動化
- スキル（`.claude/skills/`）でドメイン知識を提供
- サブエージェント（`.claude/agents/`）で専門的なタスクを委譲
