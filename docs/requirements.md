# 川津小学校PTA Webサイト リプレイス 要件定義書

## 1. 背景・目的

| 項目 | 内容 |
|---|---|
| 現行サイト | http://kawatsupta.byonia.net/ |
| 課題① | レンタルサーバー費用がかかっている |
| 課題② | HTMLを直接編集しており更新が困難（非技術者には不可） |
| 目標① | **0円運用**（サーバー費・ドメイン費ともに無料） |
| 目標② | **非技術者でも気軽に更新できる** |

---

## 2. システム構成

```
[更新担当者]
   Google ドキュメントで記事を書く
   写真はドキュメントに直接貼り付けるだけでよい（Drive保存はGASが自動処理）
        ↓
   ドキュメント内の「公開」ボタン（GASカスタムメニュー）を押す
        ↓
[GAS（Google Apps Script）]
   ・ドキュメントを Markdown に変換
   ・インライン画像 → Google Drive に自動保存 → 公開URLに置換
   ・GitHub API 経由で _posts/ に Markdown ファイルを push
        ↓
[GitHub + GitHub Pages（Jekyll組み込み）]
   push をトリガーに Jekyll が自動ビルド → 静的サイトとして公開
```

### 採用技術

| レイヤー | 技術 | 費用 |
|---|---|---|
| CMS（記事作成） | Google ドキュメント | 無料 |
| 画像置き場 | Google ドライブ（GASが自動保存・公開URL取得） | 無料（15GB） |
| 変換・デプロイスクリプト | Google Apps Script (GAS) | 無料 |
| 静的サイトジェネレータ | Jekyll（GitHub Pages 組み込み、追加設定不要） | 無料 |
| ソースコード管理・ホスティング | GitHub / GitHub Pages | 無料 |
| ドメイン | `ユーザー名.github.io/kawatsu-pta-web/` | **無料** |
| **合計** | | **0円** |

### Markdown + Jekyll を採用する理由

- **CSS・フォントが確実に反映される**: GAS が直接 HTML を生成すると Google Docs のインラインスタイルが混入して CSS が上書きされる。Markdown 経由にすれば Jekyll がクリーンな HTML を生成するため CSS が正確に適用される
- **テンプレートの一元管理**: デザイン変更は `_layouts/` の1ファイルを直すだけで全ページ・全記事に反映
- **安定性**: Markdown はシンプルで変換エラーが起きにくい

## 3. サイト構成（ページ設計）

### 現行 → 新サイト 対応表

| 現行ページ | 新サイト | 変更内容 |
|---|---|---|
| ホーム | ホーム | 更新情報一覧 + 執行部挨拶 |
| お知らせ | お知らせ | イベントレポート一覧（年別） |
| 役立ち情報 | 役立ち情報 | 内容は同等（下記参照） |
| 掲示板 | **廃止** | 静的サイトのため削除 |
| コンタクト | コンタクト | Google フォームに置き換え |

### 役立ち情報 の内訳

| コンテンツ | 方式 | 備考 |
|---|---|---|
| 行事予定 | Google ドライブ PDF or Google カレンダー埋め込み | 都度更新 |
| 広報誌「らくざん」バックナンバー | Google ドライブ限定共有リンク | 【会員限定】→ URL知る人のみ |
| 危険箇所マップ | Google ドライブ限定共有リンク（PDF） | 現行PDFをそのまま移行 |
| スポ少情報 | Google ドライブ限定共有リンク | 【会員限定】→ URL知る人のみ |
| リンク集 | 静的HTML | 学校HP、Facebook 等 |

## 4. 機能要件

### 4.1 記事投稿・更新フロー

- 担当者は Google ドキュメントで記事を執筆する
- 写真は Google ドライブ or Google フォトにアップし、ドキュメントに貼り付け or 共有リンクで参照する
- ドキュメントのカスタムメニューに「公開する」ボタンを設置する（GAS）
- ボタンを押すと自動で GitHub に反映され、数分以内に公開される
- 誤って公開しないよう、確認ダイアログを表示する

### 4.2 会員限定コンテンツ

- Google ドライブの「リンクを知っている人のみ閲覧可能」で共有する
- サイトには直接リンクを掲載する（完全非公開ではなくURLによる実質限定）
- PDFにパスワードをかけることも可（任意）

### 4.3 コンタクト

- Google フォームを作成し、サイトに埋め込み or リンクを掲載する
- 送信内容は Google スプレッドシートに自動集積される

### 4.4 掲示板

- **廃止**（静的サイトのため実装不可。必要であれば後日検討）

## 5. 非機能要件

| 要件 | 内容 |
|---|---|
| 運用コスト | **0円**（全て無料サービスで完結） |
| 更新担当者スキル | Google ドキュメントが使えれば更新可能 |
| モバイル対応 | スマートフォンで閲覧できること（レスポンシブ対応） |
| 表示速度 | 静的サイトのため高速（GitHub Pages CDN） |
| SSL | GitHub Pages が自動で HTTPS 対応 |
| バックアップ | Git 履歴が自動バックアップになる |

---

## 6. コンテンツ移行方針

| 項目 | 方針 |
|---|---|
| 新規記事 | 新サイトに投稿する |
| 既存記事（2017〜） | 移行しない（旧サイトで引き続き閲覧可能） |
| 旧サイト | しばらく並行運用し、移行が安定したら閉鎖する |
| 旧サイトアーカイブ | 閉鎖前に必要なコンテンツを Google ドライブに保存しておく |

## 9. 今後の進め方（実装ステップ案）

1. **GitHub リポジトリ作成** & GitHub Pages 設定
2. **サイトテンプレート作成**（HTML/CSS。現行サイトのデザインを参考に）
3. **GAS スクリプト作成**（Google ドキュメント → HTML 変換 → GitHub push）
4. **動作テスト**（テスト記事で一通りのフローを確認）
5. **Google フォーム作成**（コンタクト用）
6. **Google ドライブ整備**（会員限定ファイルの共有設定）
7. **公開・旧サイト並行運用開始**
8. **担当者へのマニュアル作成・引き継ぎ**
9. **旧サイト閉鎖**
